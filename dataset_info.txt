🗃️ 資料來源與目的
    本資料集原始來自 Kaggle「Bank Customer Churn Dataset」，經過資料清理與正規化處理後，拆分為多個關聯式資料表，方便進行 SQL 分析與資料查詢操作。本專案目的是建立銀行客戶流失預測的資料庫基礎結構，以利進一步分析與報表製作。

🧩 正規化後資料表一覽
| 資料表名稱   | 說明             | 欄位內容 | 
|------------- | ---------------- | ---------| 
| customer     | 客戶基本資料     | customer_id：客戶ID   | 
|               |                | gender：性別(Male,Female) |  
|                |                | age：年齡   | 
|                |                | country_id：國家ID |  
|                |                | estimated_salary：預估年薪(單位：歐元)   | 
|                |                | tenure：往來期間(單位:年) | 
| ---------------- | --------------- | ------------------------ | 
| countries      | 國家標籤資訊     | country_id：國家ID   | 
|                |                | country_name：國家名稱(France, Germany, Spain) | 
| ---------------- | --------------- | ------------------------ | 
| products       | 客戶產品資訊     | customer_id：客戶ID   | 
|                |                | products_number：擁有本行產品數量   | 
|                |                | credit_card：本行信用卡(0：無，1：有)   | 
|                |                | balance：帳戶餘額(單位：歐元) | 
| ---------------- | --------------- | ------------------------ | 
| credit_data    | 客戶信用相關資訊 | customer_id：客戶ID   | 
|                |                | credit_score：信用分數(數值範圍為0~1000，為數據集提供之模擬分數，非使用實際國際信用機構(如FICO)之分數制度。) | 
| ---------------- | --------------- | ------------------------ | 
| membership     | 客戶熱度資訊     | customer_id：客戶ID   | 
|                |                | active_member：活躍客戶(0：非活躍，1：活躍客戶)   | 
|                |                | churn：流失與否(0=未流失，1=已流失) | 
☝️ 這些表格名稱你可以依照你實際建立的 SQL 表格改寫

🔗 關聯關係（可補 ER 圖連結或文字說明）
例如：

一個 customer 對應一筆 account 資料（1 對 1）

一個 geography_id 可被多個客戶共用（1 對多）

products、salaries 均透過 customer_id 關聯回 customers

churn_status 儲存於主表（或獨立一表）並以 customer_id 關聯

📝 額外說明
原始資料中的欄位如 RowNumber, CustomerId, Surname 在正規化後被排除或重新命名

所有欄位皆已標準化命名（小寫、底線分隔）以符合 SQL 命名慣例

資料表皆已設置主鍵與適當外鍵約束
